# 🆘 Support

Thank you for using this project.

This document explains where to get help, how to report issues, and how to participate effectively in the community.

Before interacting with this repository, please review the project's
[Code of Conduct](./CODE_OF_CONDUCT.md).

---

## 📌 Support Channels

Use the appropriate channel for your situation.

### 📚 Documentation

Start with the documentation.

Documentation is usually the fastest way to find answers to common questions.

Recommended locations:

* README.md
* Documentation site
* Specifications
* Architecture documentation
* Troubleshooting guides

---

### 💬 Discussions

Use GitHub Discussions for:

* questions
* ideas
* design conversations
* architecture discussions
* community support

Discussions are preferred when:

* you are not certain a bug exists
* you want feedback on an idea
* you are exploring possible solutions

---

### 🐛 Issues

Use GitHub Issues for:

* confirmed bugs
* feature requests
* technical debt
* documentation improvements
* implementation tasks

Please search existing issues before creating a new one.

---

### 🔒 Security Issues

Do not report security vulnerabilities publicly.

Instead, follow the instructions in:

```text
SECURITY.md
```

If a security policy is not available, contact the maintainers directly.

---

## 💡 Asking Good Questions

Help us help you.

When asking for support:

* Describe what you are trying to accomplish.
* Describe what happened.
* Describe what you expected to happen.
* Include relevant logs and error messages.
* Include reproduction steps.
* Include environment details when relevant.

Examples:

* operating system
* runtime version
* package manager version
* browser version
* container image version

---

## 🦆 Troubleshooting Checklist

Before opening an issue:

* [ ] Read the documentation.
* [ ] Search existing issues.
* [ ] Search existing discussions.
* [ ] Verify you are using a supported version.
* [ ] Attempt to reproduce the issue consistently.
* [ ] Collect logs or error messages.
